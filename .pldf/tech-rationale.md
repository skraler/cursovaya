# Обоснование технологического стека

**Проект**: Визуализатор кратчайших маршрутов  
**Дата**: 21.05.2026

## Контекст решений

Исходные артефакты: [concept.md](concept.md), [ui-design.md](ui-design.md).  
Тип: одностраничное веб-приложение, визуальный редактор графа, Дейкстра по сегментам с промежуточными точками, анимация «Пуск».

---

## 1. JavaScript в браузере (без Python)

### Варианты

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **Только JS** | Один runtime, мгновенный отклик UI, простой запуск | Алгоритм нужно писать на JS |
| **JS + Python API** | Привычный Python для алгоритмов | Два процесса, HTTP, CORS, сложнее защита и отчёт |

### Решение

**Только JavaScript.** Для графа до ~50 вершин Дейкстра на клиенте выполняется за миллисекунды. Python имел бы смысл при серверном хранении, многопользовательском режиме или требовании методички — ни то, ни другое не задано.

---

## 2. Vanilla JavaScript + Vite

### Варианты

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **HTML + script без сборки** | Нулевая настройка | Слабая модульность, неудобные тесты |
| **Vite + Vanilla JS** | ES-модули, `npm run dev`, Vitest из коробки | Нужен Node.js |
| **React + Vite** | Компоненты, экосистема | Лишняя абстракция для одного экрана |

### Решение

**Vite + Vanilla JS.** Прототип уже свёрстан на семантическом HTML/SVG; перенос в `src/ui` естественен. React увеличил бы объём курсовой без пользы для задачи «нарисовать граф и подсветить путь».

**Версии (ориентир при инициализации проекта):**
- `vite` ^6.x
- `vitest` ^3.x

---

## 3. Отрисовка: SVG

### Варианты

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **SVG** | DOM-события на вершинах/рёбрах, CSS-анимация stroke | Много элементов при тысячах узлов |
| **Canvas** | Быстрее при очень больших графах | Сложнее hit-test и подписи весов |

### Решение

**SVG** — согласовано с прототипом; клики по вершинам, подписи весов, плавная зелёная анимация через CSS `transition` на `stroke`.

---

## 4. Vitest для тестов

### Варианты

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **Без тестов** | Быстрее сдать | Риск ошибок в Дейкстре на защите |
| **Vitest** | Те же модули, что в приложении; быстрый запуск | Нужно написать 5–15 тестов |
| **Jest** | Популярен | Тяжелее интеграция с Vite, чем Vitest |

### Решение

**Vitest** — студент выбрал вариант с тестами. Покрытие: корректность Дейкстры, недостижимая вершина, неориентированное ребро как две directed, склейка сегментов маршрута A→B→C.

---

## 5. Организация кода: Layer-first

### Варианты

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| **Layer-first** | Чистая логика без DOM; удобно для курсовой «алгоритм + интерфейс» | Фича размазана по папкам |
| **Feature-first** | Всё про «маршрут» в одной папке | Дублирование graph в фичах |

### Решение

**Layer-first** — выбор студента. Слои:
- `graph` — не зависит от UI;
- `algorithms` — только граф и числа;
- `state` — связывает модель и режимы инструментов;
- `ui` — SVG и обработчики.

---

## 6. Что сознательно не входит в стек

| Технология | Причина исключения |
|------------|-------------------|
| PostgreSQL / SQLite | Нет персистентности в scope |
| Flask / FastAPI | Нет backend |
| React / Vue | Нет необходимости |
| WebSocket | Нет real-time |
| TypeScript | Не выбрано; можно добавить позже |

---

## Совместимость стека

```
Vite (build) → статический bundle → браузер
Vitest → тестирует graph/ + algorithms/ (node environment)
ui/ → импортирует state/, state/ → graph/, algorithms/
```

Конфликтов зависимостей нет: единая экосистема JavaScript.
