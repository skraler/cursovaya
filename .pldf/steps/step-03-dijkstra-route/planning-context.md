## Почему этот шаг создан

После graph/ логично реализовать ядро курсовой — алгоритм Дейкстры и сборку маршрута с промежуточными точками (ключевое решение концепции). Без этого Store и UI нечего показывать.

## Зависимости

- **Зависит от**: `step-02-graph-model`

- **Покрывает P1**:
  - Расчёт кратчайшего пути (алгоритм Дейкстры по сегментам)
  - Обработка недостижимости сегмента
  - Подготовка данных для отображения результата (RouteResult)

## Ключевые решения

### РЕШЕНИЕ: Отдельные dijkstra и routePlanner

- **Причина**: Дейкстру тестируют изолированно; routePlanner — оркестрация сегментов
- **Альтернатива**: один файл — отклонена для читаемости отчёта

### РЕШЕНИЕ: Приоритетная очередь (опционально упрощённая)

- **Причина**: до 50 вершин; можно O(V²) для простоты курсовой на «3»
- **Влияние**: в отчёте указать сложность
