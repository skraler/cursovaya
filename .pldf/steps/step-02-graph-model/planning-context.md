# Контекст планирования: Шаг 02 - Модель графа

**Шаг**: step-02-graph-model  
**Дата планирования**: 2026-05-21  
**Запланирован на итерации планирования**: 2

## Почему этот шаг создан

После setup нужен доменный слой графа — без него нельзя писать Дейкстру и Store. Выбран по приоритету P1 и зависимостям: единственный шаг, готовый после step-01.

## Зависимости

- **Зависит от шагов**:
  - `step-01-project-setup`: Vite, Vitest, каталоги `src/graph/`

- **Покрывает функции из концепции**:
  - **P1**: Режимы графа (ориентированное / неориентированное ребро)
  - **P1** (частично): основа для визуального редактора графа (CRUD на уровне данных)

## Ключевые решения при планировании

### РЕШЕНИЕ: Список смежности внутри Graph

- **Дата**: 2026-05-21
- **Причина**: выбор студента на этапе architecture; стандарт для Дейкстры
- **Альтернативы**:
  - **Матрица смежности** — не выбрана: избыточна для разреженных графов
- **Влияние**: step-03 dijkstra использует getNeighbors / adjacency

### РЕШЕНИЕ: Одна сущность Edge + две adjacency-записи для undirected

- **Причина**: data-model.md — одно ребро в UI, двунаправленная связь в алгоритме
- **Влияние**: removeEdge и подсветка маршрута по edgeId
