# Концепция проекта: Визуализатор кратчайших маршрутов

**Дата создания**: 21.05.2026

## Целевая аудитория

### Основные пользователи

Любой пользователь, которому нужно спланировать маршрут на условной схеме или карте: логистика, навигация по объекту, учебные и демонстрационные задачи. Пользователь не обязан знать теорию графов — он работает с наглядной схемой точек и связей.

**Портрет пользователя:**
- Планирует путь между несколькими точками на схеме (склад, офис, перекрёстки, этажи здания и т.п.)
- Хочет видеть результат на экране, а не считать вручную
- Использует браузер на компьютере или ноутбуке

### Потребности и ожидания

- Быстро нарисовать граф без ввода матриц и формул
- Задать начало, конец и промежуточные остановки в нужном порядке
- Получить кратчайший маршрут с понятной длиной (суммой весов)
- Понимать, по каким рёбрам идёт путь (подсветка на схеме)

### Сценарии использования

1. **Маршрут с обязательными остановками** — пользователь рисует граф, выбирает «Старт → точка A → точка B → Финиш», приложение строит кратчайший путь по каждому сегменту и показывает полный маршрут.
2. **Схема объекта** — пользователь расставляет вершины как комнаты/узлы, соединяет коридоры весами (расстояние/время), проверяет оптимальный проход.
3. **Демонстрация маршрута** — пользователь наглядно проверяет работу алгоритма Дейкстры на произвольном графе.

## Решаемая "боль" целевой аудитории

**Проблема:** при нескольких обязательных остановках кратчайший путь сложно посчитать вручную — высок риск ошибки; таблица чисел не показывает, как выглядит маршрут на схеме.

**Кто испытывает:** пользователи, планирующие маршруты на графовых моделях (схемы, сети, условные карты).

**Как решает проект:** веб-приложение с визуальным редактором графа и автоматическим расчётом кратчайшего маршрута алгоритмом Дейкстры (по сегментам между последовательными точками маршрута) с подсветкой пути и отображением суммарной длины.

## Основные функции разрабатываемого проекта

### Приоритет 1 (P1) — В первую очередь

- **Визуальный редактор графа**: добавление вершин кликом на холсте, создание рёбер между вершинами, задание веса ребра.
- **Режимы графа**: при создании ребра пользователь указывает — ориентированное оно или неориентированное (связь в обе стороны).
- **Выбор маршрута**: назначение вершины «Старт», «Финиш» и упорядоченного списка обязательных промежуточных точек.
- **Расчёт кратчайшего пути**: для каждой пары соседних точек маршрута (Старт→1, 1→2, …, N→Финиш) — алгоритм Дейкстры; склейка сегментов в полный маршрут.
- **Отображение результата**: подсветка рёбер/вершин маршрута на холсте, показ суммарной длины маршрута.
- **Обработка недостижимости**: сообщение пользователю, если между двумя точками сегмента нет пути.

### Приоритет 2 (P2) — После P1

- **Пошаговая анимация** работы Дейкстры (для отчёта и защиты курсовой).
- **Редактирование графа**: удаление/перемещение вершин и рёбер, изменение весов.
- **Сброс и очистка** холста одним действием.

### Приоритет 3 (P3) — Функции последней очереди

- **Сохранение и загрузка** графа в файл (JSON или аналог).
- **Готовые примеры графов** (шаблоны для быстрого старта).
- **Экспорт** изображения схемы с маршрутом для отчёта.

## Ограничения и предположения

### Технические ограничения

- Платформа: **веб-приложение** в браузере (без установки на ПК).
- Веса рёбер **неотрицательные** (алгоритм Дейкстры не работает с отрицательными весами).
- Объём графа для курсовой: ориентир до ~30–50 вершин (достаточно для демонстрации, без требований промышленного масштаба).
- P1 без серверной части: расчёт и интерфейс на стороне клиента (достаточно для оценки «3»).

### Предположения

- **Пользователь** умеет работать с мышью в браузере и понимает идею «точка — связь — расстояние».
- **Граф связный** в пределах выбранного маршрута; при разрыве пользователь получает понятное сообщение.
- **Промежуточные точки** обязательны и проходятся **строго в заданном порядке** (не переставляются алгоритмом).
- **Один пользователь** за сеанс, без совместного редактирования и учётных записей (на этапе P1).

## Ключевые решения этапа

### 1. Маршрут с обязательными промежуточными точками по сегментам

- **Причина**: пользователю нужен путь «Старт → остановка 1 → … → Финиш»; один запуск Дейкстры от старта до финиша не гарантирует прохождение всех остановок, поэтому маршрут собирается из кратчайших сегментов между соседними точками в заданном порядке.
- **Влияние**: на архитектуру (модуль сегментов + склейка пути), UI (очередь выбора точек), план реализации (отдельные шаги: граф, выбор точек, Дейкстра, визуализация).

### 2. Веб-платформа с визуальным редактором на холсте

- **Причина**: наглядность — ключевая ценность продукта; браузер упрощает демонстрацию на защите без установки ПО.
- **Влияние**: на design (макет холста, панель инструментов), tech (HTML/CSS/JS или фреймворк с canvas/SVG), deploy (статический хостинг).

### 3. P1-минимум без анимации и сохранения

- **Причина**: цель — рабочая курсовая на оценку «3» в разумные сроки; ядро — граф, расчёт, подсветка и длина маршрута.
- **Влияние**: на implementation-plan (сжатый MVP), отложение P2/P3 до основной сдачи.
